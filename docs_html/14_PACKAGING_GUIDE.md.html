CurseForge 預算指南

本文件定義 EventAlertMod Retail rewrite 的發佈資源規則。

快速指令

日後用戶只需輸入：

打包

就執行：

powershell -ExecutionPolicy Bypass -File .\Tools\Build-CurseForgePackage.ps1

或使用者只需輸入：

打包開發版

就執行：

powershell -ExecutionPolicy Bypass -File .\Tools\Build-CurseForgePackage.ps1 -DevMode

除非使用者附加指定參數，否則不需要再次詢問。

版本命名

TOC 版本名稱必須符合：

EventAlertMod_資料片簡稱_打包年月日

目前資料片名稱固定為：

MN

範例：

## Version: EventAlertMod_MN_20260504

備用檔名

zip檔名必須符合：

EventAlertMod_資料片簡稱_打包年月日_打包時分秒.zip

範例：

EventAlertMod_MN_20260504_205216.zip

預設排除內容

可選參數

包含文件與 AGENTS：

powershell -ExecutionPolicy Bypass -File .\Tools\Build-CurseForgePackage.ps1 -IncludeDocs

跳過Lua語法檢查：

powershell -ExecutionPolicy Bypass -File .\Tools\Build-CurseForgePackage.ps1 -SkipLuaCheck

開發版本壓縮（將整個專案資料壓縮，不過濾，排除 .git/ 和 Dist/）：

powershell -ExecutionPolicy Bypass -File .\Tools\Build-CurseForgePackage.ps1 -DevMode

驗證要求

預算流程必須完成：

  • TOC 路徑驗證。
  • Lua 語法檢查，除非明確使用 -SkipLuaCheck。
  • zip排除檢查，確認沒有legacy/reference/tools/backup/dev資料夾。

HTML 說明檔案轉換

  • 對於 Docs/ 下或根目錄 AGENTS.md 涉及表格、圖像、心智圖與流程圖的 Markdown 文件，應執行 HTML 轉換工具，在 docs_html/ 底下產生一個同名的 .html 文件（如 __10）。
  • 開發與AI自動協作時，一律以.md檔案為絕對的Facts-of-Truth參考。 .html版本供人類好讀與預覽使用，不可被自動化AI開發的配置或方案碼邏輯之引用。

注意事項

  • 資源成功不代表WoW正式服實機驗證完成。
  • 若 TOC ## Version 與當日日期不一致，壓縮工具會停止。
  • 發布前仍需依Docs/06_TEST_PLAN_RETAIL.md做正式服實機測試。

CurseForge Budget Guide

This file defines publishing resource rules for EventAlertMod Retail rewrite.

Quick instructions

In the future, users only need to enter:

打包

Just execute:

powershell -ExecutionPolicy Bypass -File .\Tools\Build-CurseForgePackage.ps1

Or the user can simply enter:

打包開發版

Just execute:

powershell -ExecutionPolicy Bypass -File .\Tools\Build-CurseForgePackage.ps1 -DevMode

Unless the user appends specific parameters, there is no need to ask again.

Version naming

TOC version name must match:

EventAlertMod_資料片簡稱_打包年月日

The current expansion pack name is fixed at:

MN

example:

## Version: EventAlertMod_MN_20260504

Alternate file name

The zip file name must match:

EventAlertMod_資料片簡稱_打包年月日_打包時分秒.zip

example:

EventAlertMod_MN_20260504_205216.zip

Default default content

  • EventAlertMod.toc
  • README.md
  • readme.html
  • changelog.txt
  • Core/
  • Services/
  • UI/
  • Debug/
  • Data/
  • Locale/
  • media/

Default excluded content

  • LegacyReference/
  • ReferenceLibs/
  • Tools/
  • Backup/
  • .github/
  • .vscode/
  • documents/
  • Dist/_stage/

Optional parameters

Include files with AGENTS:

powershell -ExecutionPolicy Bypass -File .\Tools\Build-CurseForgePackage.ps1 -IncludeDocs

Skip Lua syntax checking:

powershell -ExecutionPolicy Bypass -File .\Tools\Build-CurseForgePackage.ps1 -SkipLuaCheck

Development version compression (compresses the entire project data, does not filter, excludes .git/ and Dist/):

powershell -ExecutionPolicy Bypass -File .\Tools\Build-CurseForgePackage.ps1 -DevMode

Verification requirements

The budget process must be completed:

  • TOC path verification.
  • Lua syntax checking unless -SkipLuaCheck is used explicitly.
  • zip exclusion check to confirm that there is no legacy/reference/tools/backup/dev folder.

HTML Description file conversion

  • For Markdown files under Docs/ or the root directory AGENTS.md involving tables, images, mind maps and flow charts, the HTML conversion tool should be executed to generate a .html file with the same name (such as __10) under docs_html/.
  • When developing automatic collaboration with AI, always use the .md file as the absolute Facts-of-Truth reference. The .html version is for human reading and preview use and cannot be referenced by the configuration or program code logic of automated AI development.

Notes

  • The successful resource does not mean that the official WoW server verification is completed.
  • If TOC ## Version is inconsistent with the current date, the compression tool will stop.
  • Retail server testing still needs to be done in accordance with Docs/06_TEST_PLAN_RETAIL.md before release.
