正式服測試計劃

此文件通行證未執行任何即時正式服驗證。

靜態檢查

  • 確認僅正式服 TOC 處於活動狀態以進行重寫。
  • 當針對 Retail 12.0.7 時，確認 TOC 介面為 120007。
  • 確認未載入 Classic/MOP/Cata/Wrath/TBC 來源根。
  • 搜尋意外的全域變數。
  • 搜尋 C_Timer.After(function.
  • 搜尋每幀/per-icon SetScript("OnUpdate")。
  • 搜尋大型項目 ID 掃描循環。
  • 搜尋應用於 SavedVariables 或運行時狀態的 table.freeze。
  • 搜尋舊版 UnitAura 解包路徑。
  • 搜尋舊的全域 GetSpellCooldown 解包假設。
  • 在秘密檢查之前搜尋不安全光環spellID比較。
  • 搜尋用作事實的 Cooldown 小部件 getter 讀回。
  • 搜尋戰鬥中框架創建或佈局突變。

登入 / ReloadUI

  • 在 Retail 12.x 中載入插件。
  • 確認登入時沒有 Lua 錯誤。
  • 確認SavedVariables遷移一次並保持可寫入。
  • 確認「/reload」保留設定檔和位置。
  • 確認已停用的警報保持停用狀態。

斜線指令

  • /eam opt 開啟選項。
  • 在影格存在之前在戰鬥中呼叫的「/eam opt」不應建立受保護的操作或污點錯誤。
  • /eam help 列印指令摘要。
  • /eam add <spellID> 新增玩家光環警報。
  • /eam add target <spellID> 新增目標光環警報。
  • /eam add cd <spellID> 新增法術冷卻時間警報。
  • /eam add item <itemID> 新增項目冷卻警報。
  • /eam remove <spellID> 刪除玩家光環警報。
  • /eam 刪除目標 <spellID> 刪除目標光環警報。
  • /eam remove cd <spellID> 刪除法術冷卻時間警報。
  • /eam 刪除物品 <itemID> 刪除物品冷卻警報。
  • /eam export 列印緊湊的提示/debug export。
  • /eam show 切換自我光環 spellID 偵測。
  • /eam showt 切換目標光環 spellID 偵測。
  • /eam showc 切換強制轉換 spellID 偵測。
  • /eam showa 是可選的並且顯然是可停止的。
  • /eam MiniMap 切換小地圖選項按鈕。
  • /eam MiniMap Reset 重設小地圖位置。
  • /eam SCDRemoveWhenCooldown
  • /eam SCDNocombatStillKeep
  • /eam SCDGlowWhenUsable
  • /eam IconAppenSpellTip
  • /eam ShowRunesBar
  • name/timer/stack 文字的字體大小指令。
  • 新的調試導出命令僅根據需要產生緊湊的輸出。

選項 UI 測試

  • 使用「/eam opt」開啟脫離戰鬥的選項面板。
  • 確認數字 ID 編輯框僅接受數字。
  • 從面板中新增/remove玩家光環、目標光環、法術冷卻時間和物品冷卻時間條目。
  • 確認每個成功的按鈕操作都會增加 SavedVariables 修訂版並刷新符合的服務。
  • 確認無效或空 ID 顯示簡短的狀態訊息，而不會引發 Lua 錯誤。
  • 確認關閉並重新開啟面板不會重複重新建立小工具。
  • 在建立面板之前在戰鬥中打開“/eam opt”，並確認EAM列印一條延遲訊息而不是建構幀。
  • 在面板已經存在後，在戰鬥中開啟“/eam opt”，並確認 show/hide 不會污染安全 UI 路徑。

玩家光環測試

  • 透過 spellID 新增玩家增益並出現確認圖示。
  • 透過「/eam add 」、「/reload」加入玩家增益，並確認其持續存在。
  • 透過/eam刪除<spellID>刪除該buff並確認圖示隱藏。
  • 透過 spellID 新增玩家減益並出現確認圖示。
  • 確認堆疊計數更新。
  • 確認計時器文字僅在安全時出現。
  • 當duration/expiration不可用時，確認圖示保持穩定。
  • 確認player/pet符合行為符合設定。
  • 確認空閒時沒有高頻全掃描。
  • 確認 UNIT_AURA 增量更新過程 addedAuras、updatedAuraInstanceIDs 和 removedAuraInstanceIDs。
  • 當 updateInfo 為零或標記為完全更新時，確認完全更新回退仍然有效。
  • 確認完整更新回退會針對每個相關過濾器掃描一次設備，並且不會針對每個配置的警報掃描一次。
  • 確認完整掃描中未配置的光環警報被標記為非活動狀態。

目標光環測試

  • 透過 spellID 新增目標 buff/debuff。
  • 透過/eam新增目標<spellID>和/eam刪除目標<spellID>新增/remove目標光環。
  • 快速改變目標。
  • 明確的目標。
  • 確認目標清除將所有配置的目標光環警報標記為非活動狀態。
  • 進入/leave戰鬥並啟動目標警報。
  • 確認陳舊的目標圖示已被刪除或標記為非活動狀態。
  • 確認自身減益過濾有效。

法術冷卻測試

  • 透過 spellID 加入法術冷卻時間。
  • 施展法術並確認冷卻圖示/timer。
  • 確認基於衝鋒的法術的衝鋒更新。
  • 確認 GCD-only 冷卻時間不會錯誤地顯示為真實冷卻時間。
  • 確認安全時可用的發光符合 C_Spell.IsSpellUsable。
  • 確認沒有按法術計時器攪拌。

物品冷卻測試

  • 增加直接 itemID 冷卻時間。
  • 如果支持，請使用裝備的物品和庫存物品。
  • 確認物品冷卻事件刷新。
  • 確認預設未建置物品法術快取。
  • 在戰鬥之外開始可選的快取建置。
  • 確認快取在戰鬥中和低 FPS 下暫停。

戰鬥測試

  • 使用主動 self/target/cooldown 警報進入戰鬥。
  • 確認沒有受保護的操作錯誤。
  • 確認 EAM 訊框更新不會產生污染/阻止操作錯誤。
  • 確認不安全的資料會降級而不是崩潰。
  • 確認如果池在戰鬥中耗盡，則首次圖示創建將被推遲。
  • 確認延遲版面配置在 PLAYER_REGEN_ENABLED 之後刷新。
  • 確認戰鬥中沒有開始大量緩存建置。
  • 確認戰鬥結束後運行非戰鬥刷新。

正式服 12.0.7/午夜 API 測試

  • 確認 C_DurationUtil.CreateDurationTextBinding 存在並確定它是否有利於 EAM 計時器標籤。
  • 2026-05-29 PTR 注意：使用者確認最小 C_DurationUtil.CreateDurationTextBinding 範例在 12.0.7 PTR 用戶端中正常顯示。
  • 確認C_DurationUtil.CreateManualClock存在且不需要不安全的Lua倒數使用。
  • 確認 EAM 不會呼叫已刪除的 C_DurationUtil.GetCurrentTime。
  • 確認 GetEventCPUUsage、GetFunctionCPUUsage 和 GetScriptCPUUsage 僅適用於 debug/profiling 指令，不適用於執行時間熱路徑。
  • 確認靜態表上的 table.freeze / table.isfrozen 行為，並驗證 SavedVariables/runtime 狀態沒有被凍結。
  • 確認「冷卻：SetCooldownFromDurationObject()」適用於 EAM 僅顯示計時器狀態。
  • 確認 FontString:ClearText() 清除文本，沒有污染或過時的秘密文本問題。
  • 確認未來的 EAM DurationTextBinding 適配器保留綁定引用，在回收圖示時停用 /releases 它，並在 API 不可用時安全回退。

本地化測試

  • enUS 載入。
  • zhTW 僅載入並包含繁體中文字串。
  • zhCN/koKR/ruRU 舊字串如果保留，將保持隔離。
  • 遺失的語言環境字串安全回退。

偵錯匯出測試

  • debug-min 輸出包括環境、事實、衍生計數和警告。
  • analysis-full 輸出包含緊湊的每個警報狀態。
  • github-issue 輸出排除大量日誌和敏感的僅限本地的混亂。
  • 導出不會自動運作。
  • /eam 匯出 包含資料庫修訂、aura 快取計數、渲染器可見/deferred 計數和邊界警告計數。

Retail server test plan

This file pass does not perform any live server verification.

Static check

  • Confirmed that only production server TOC is active for rewrites.
  • Confirmed TOC interface is 120007 when targeting Retail 12.0.7.
  • Verify that the Classic/MOP/Cata/Wrath/TBC source root is not loaded.
  • Search for unexpected global variables.
  • Search for C_Timer.After(function.
  • Search for /per-icon SetScript("OnUpdate") per frame.
  • Search for large project ID scan loops.
  • Search for table.freeze applied to SavedVariables or runtime state.
  • Search for old UnitAura unpack paths.
  • Search the old global GetSpellCooldown unpacking hypothesis.
  • Search for unsafe Aura before secret inspection spellID comparison.
  • Search for the Cooldown widget getter used as a fact readback.
  • Search for frame creation or layout mutations in combat.

Login / ReloadUI

  • Load plugin in Retail 12.x.
  • Verify there are no Lua errors when logging in.
  • Confirm that SavedVariables are migrated once and remain writable.
  • Confirm that "/reload" retains the profile and location.
  • Confirm that deactivated alerts remain deactivated.

slash command

  • /eam opt enables options.
  • "/eam opt" called in combat before the shadow frame exists should not create protected operations or taint errors.
  • /eam help prints a summary of the command.
  • /eam add <spellID> Added player Aura alert.
  • /eam add target <spellID> Added target Aura alert.
  • /eam add cd <spellID> Added spell cooldown time alert.
  • /eam add item <itemID> Added item cooling alert.
  • /eam remove <spellID> Removes player Aura alerts.
  • /eam Remove target <spellID> Remove target Aura alert.
  • /eam remove cd <spellID> Removes the spell cooldown alert.
  • /eam Remove Item <itemID> Remove item cooldown alert.
  • /eam export prints the compact prompt /debug export.
  • /eam show toggles self-Aura spellID detection.
  • /eam showt toggle target Aura spellID detection.
  • /eam showc toggles detection of cast spellID.
  • /eam showa is optional and apparently stoppable.
  • /eam MiniMap toggle minimap option buttons.
  • /eam MiniMap Reset Reset the minimap position.
  • /eam SCDRemoveWhenCooldown
  • /eam SCDNocombatStillKeep
  • /eam SCDGlowWhenUsable
  • /eam IconAppenSpellTip
  • /eam ShowRunesBar
  • name/timer/stack font size directive for text.
  • New debug export command produces compact output only on demand.

Options UI Test

  • Use "/eam opt" to open the options panel for leaving combat.
  • Confirm that the numeric ID edit box only accepts numbers.
  • Added/remove player Aura, target Aura, spell cooldown and item cooldown entries from the panel.
  • Confirm that each successful button operation increments the SavedVariables revision and refreshes the conforming service.
  • Confirmation of invalid or empty IDs displays a brief status message without raising a Lua error.
  • Make sure closing and reopening the panel does not re-create the gadget again.
  • Turn on "/eam opt" in combat before creating the panel and make sure EAM prints a delay message instead of building a frame.
  • Turn on "/eam opt" in combat after the panel already exists, and confirm that show/hide does not pollute the safe UI path.

Player Halo Test

  • Add player buffs via spellID and a confirmation icon will appear.
  • Add player buffs through "/eam add " and "/reload" and confirm their continued existence.
  • Delete the buff via /eam delete <spellID> and confirm that the icon is hidden.
  • Add player debuff via spellID and a confirmation icon will appear.
  • Confirm stack count update.
  • Confirmation timer text only appears when safe.
  • Make sure the icon remains stable when duration/expiration is not available.
  • Confirm that the player/pet complies with the behavior and settings.
  • Confirmed no high frequency full scan when idle.
  • Confirm UNIT_AURA incremental update process addedAuras, updatedAuraInstanceIDs and removedAuraInstanceIDs.
  • Confirm that the full update fallback still works when updateInfo is zero or marked as a full update.
  • Confirmed that full update fallback will scan the device once for each relevant filter and not once for each configured alert.
  • Confirmed that unconfigured halo alerts in full scans are marked as inactive.

Target Aura test

  • Added target buff/debuff via spellID.
  • Add/remove target Aura through /eam add target <spellID> and /eam remove target <spellID>.
  • Quickly change goals.
  • Clear goals.
  • Confirm Target Clear marks all configured target Aura alerts as inactive.
  • Enter /leave combat and activate target alert.
  • Confirm that stale target icons have been removed or marked as inactive.
  • Confirm that self-debuff filtering is effective.

Spell Cooldown Test

  • Added spell cooldown via spellID.
  • Cast the spell and confirm the cooldown icon /timer.
  • Confirmed charge update for charge based spells.
  • Confirmed that GCD-only cooldowns are not incorrectly displayed as real cooldowns.
  • The glow available when safe is confirmed to comply with C_Spell.IsSpellUsable.
  • Make sure you are not stirring on the spell timer.

Item Cooldown Test

  • Added direct itemID cooldown.
  • Use equipped items and inventory items if supported.
  • Confirm item cooldown event refresh.
  • Confirmed that item spell cache is not built by default.
  • Start optional cache building outside of combat.
  • Confirm cache is paused in combat and at low FPS.

Combat Test

  • Enter combat with active self/target/cooldown alert.
  • Confirm that there are no protected operation errors.
  • Confirmed that EAM frame updates do not produce taint/blocking operation errors.
  • Confirmed that insecure profiles will degrade rather than crash.
  • Confirmed that first icon creation will be delayed if the pool is depleted during combat.
  • Confirm deferred layout configuration refreshes after PLAYER_REGEN_ENABLED.
  • Confirmed that mass cache building does not start during combat.
  • Run non-combat refresh after confirming the end of the battle.

Retail server 12.0.7/Midnight API test

  • Verify that C_DurationUtil.CreateDurationTextBinding exists and determine if it favors the EAM timer tag.
  • 2026-05-29 PTR Note: User confirmed minimum C_DurationUtil.CreateDurationTextBinding examples display properly in the 12.0.7 PTR client.
  • Confirm that C_DurationUtil.CreateManualClock exists and does not require unsafe Lua reciprocal usage.
  • Confirmed that EAM does not call the deleted C_DurationUtil.GetCurrentTime.
  • Confirmed that GetEventCPUUsage, GetFunctionCPUUsage and GetScriptCPUUsage only apply to debug/profiling instructions and not to execution-time hot paths.
  • Confirm table.freeze / table.isfrozen behavior on static tables, and verify that SavedVariables/runtime state is not frozen.
  • Confirmed that "Cooldown:SetCooldownFromDurationObject()" works for EAM only showing timer status.
  • Confirm that FontString:ClearText() cleans up the text and has no Taint or outdated secret text issues.
  • Confirm that future EAM DurationTextBinding adapters retain binding references, deactivate /releases when recycling icons, and safely fall back when API is unavailable.

Localization testing

  • enUS loading.
  • zhTW only loads and contains Traditional Chinese strings.
  • zhCN/koKR/ruRU The old string, if retained, will remain isolated.
  • Safe fallback for missing locale strings.

Debug export test

  • debug-min Output includes environment, facts, derived counts, and warnings.
  • analysis-full output contains a compact representation of each alarm status.
  • github-issue output excludes bulk logging and sensitive local-only clutter.
