EAM 實作導向摘要

WoW API

WoW API 是主要 runtime API 入口。EAM rewrite 應優先從 API system 找到對應 namespace，而不是保留舊 global wrapper。

重要三層：

WoW API
├─ API systems
│  ├─ C_UnitAuras
│  │  └─ AuraService
│  ├─ C_Spell
│  │  └─ CooldownService / SpellInfoService
│  ├─ C_Item
│  │  └─ ItemCooldownService
│  ├─ C_TooltipInfo
│  │  └─ low-frequency fallback only
│  └─ C_Secrets
│     └─ boundary checks

結論：

  • 12.x 只用 Retail C_* namespace。
  • C_Secrets 應集中封裝，不要散落在每個 UI function。
  • C_TooltipInfo 是 fallback 資料來源，不是 hot path 事實來源。

Lua API

Lua API 說明 WoW 使用 Lua 5.1 子集，且缺少 OS / file I/O library。

重要三層：

Lua API
├─ basic functions
│  ├─ pcall / xpcall
│  └─ type / select
├─ string
│  ├─ string.match / gsub / format
│  └─ WoW custom string helpers
├─ table
│  ├─ table.create
│  ├─ table.wipe
│  ├─ table.freeze
│  └─ table.isfrozen

結論：

  • string.match 不可用在未確認安全的 secret-derived tooltip text。
  • hot path 避免 transient table、table.insert、大量 pairs。
  • table.freeze 只用於 immutable constants。

FrameXML API

FrameXML functions 包含 UI helper、pool、EventUtil、TooltipUtil、LinkUtil 等。

重要三層：

FrameXML API
├─ Mixins / pools
│  ├─ CreateFramePool
│  ├─ CreateTexturePool
│  └─ CreateFontStringPool
├─ SharedXML
│  ├─ EventUtil
│  ├─ FrameUtil
│  └─ TooltipUtil
└─ UIParent / panels
   ├─ ShowUIPanel
   └─ Toggle*Frame

結論：

  • EAM 的 UI/IconPool.lua 可以參考 Blizzard pool pattern。
  • 不要在 combat 中呼叫會開關 protected panel 的函式。
  • TooltipUtil 可作參考，但 tooltip parsing 必須低頻且 secret-safe。

Widget API / Widget Scripts

Widget API 與 Widget script handlers 是 EAM Renderer、EventRouter、Scheduler 的基礎。

重要三層：

Widget API
├─ Frame
│  ├─ RegisterEvent
│  ├─ RegisterAllEvents
│  └─ SetScript
├─ Cooldown
│  ├─ SetCooldownFromDurationObject
│  ├─ SetCountdownFormatter
│  └─ SetCountdownMillisecondsThreshold
└─ FontString
   ├─ SetText
   └─ ClearText

Widget scripts
├─ Frame
│  ├─ OnEvent
│  └─ OnUpdate
├─ Button
│  ├─ OnClick
│  └─ PreClick / PostClick
└─ EditBox
   ├─ OnTextChanged
   └─ OnEnterPressed / OnEscapePressed

結論：

  • RegisterAllEvents 只適合 debug/datamining，不適合 EAM runtime。
  • EAM 只應有一個 EventRouter OnEvent。
  • EAM 只應有一個 Scheduler OnUpdate。
  • Cooldown 應承接 DurationObject，不要讓 Lua 每幀算秒。

XML schema

XML schema 說明 XML 可以建立 UI，但 Lua Widget API 也能做多數事情。

重要三層：

XML schema
├─ Ui
│  ├─ Include
│  └─ Script
├─ LayoutFrame
│  ├─ Size
│  ├─ Anchors
│  └─ Animations
└─ Frame
   ├─ Layers
   ├─ Frames
   └─ Scripts

結論：

  • 新架構可保留簡單 XML loader，但 runtime icons 應交給 IconPool。
  • 避免 XML 內建立大量 global frame name。
  • 避免 XML OnUpdate。

Events

Events / Category:API events 是 EventRouter 的事件來源。

重要三層：

Events
├─ UNIT_AURA
│  ├─ unitTarget
│  ├─ updateInfo.addedAuras
│  ├─ updateInfo.updatedAuraInstanceIDs
│  └─ updateInfo.removedAuraInstanceIDs
├─ cooldown events
│  ├─ SPELL_UPDATE_COOLDOWN
│  ├─ SPELL_UPDATE_CHARGES
│  └─ BAG_UPDATE_COOLDOWN
└─ combat boundary
   ├─ PLAYER_REGEN_DISABLED
   └─ PLAYER_REGEN_ENABLED

結論：

  • UNIT_AURA 增量資料是 AuraService 核心入口。
  • removed aura instance ID 無法再查完整 aura，服務要先 cache 安全摘要。
  • 脫戰刷新是 boundary-limited state 的校準點。

CVars

Console variables 說明 CVar 可以用 GetCVar / SetCVar / /console 管理，部分 secure CVar 不能在 combat 改。

重要三層：

CVars
├─ query
│  ├─ GetCVar
│  ├─ GetCVarDefault
│  └─ C_CVar.GetCVarInfo
├─ mutation
│  ├─ SetCVar
│  └─ /console
└─ reset
   ├─ cvar_default
   └─ cvar_reset

結論：

  • EAM 不應依賴改 CVar 達成功能。
  • secret forced CVars 只應放 live validation checklist。
  • 不在 combat 中修改 secure CVar。

Macro Commands

Macro commands 分為 character、combat 等指令。

重要三層：

Macro commands
├─ character commands
├─ combat commands
│  ├─ cast / use
│  ├─ cancelaura
│  └─ startattack / stopattack
└─ addon slash commands
   ├─ /eam opt
   ├─ /eam debug
   └─ /eam export

結論：

  • EAM slash commands 只能做設定、偵測、debug、匯出。
  • 不新增 combat automation。
  • 不產生 cast/use/cancelaura 類行為。

Combat Log

Combat Log / COMBAT_LOG_EVENT_UNFILTERED 是歷史上常見 AddOn 資料來源，但 12.x 目標與 EAM rewrite 不應依賴盲掃。

重要三層：

Combat Log
├─ Combat Log window / file
├─ COMBAT_LOG_EVENT_UNFILTERED
│  ├─ source
│  ├─ destination
│  ├─ prefix
│  └─ suffix
└─ EAM boundary
   ├─ not primary source
   ├─ no heavy parsing
   └─ use event services instead

結論：

  • 12.x 的 EAM 不以 CLEU 作為 aura/cooldown 主要來源。
  • 只在必要 debug 或非 hot path 檢查時參考。
  • 不建立 WeakAura-like combat parser。

Escape Sequences / Hyperlinks

UI escape sequences 和 Hyperlinks 是 debug text、tooltip、import/export 的風險區。

重要三層：

Escape sequences
├─ color
├─ texture
└─ reset

Hyperlinks
├─ link format
│  ├─ color prefix
│  ├─ type:payload
│  └─ display text
├─ inspection
│  ├─ SetItemRef
│  └─ ExtractHyperlinkString
└─ EAM use
   ├─ spell link display
   ├─ item link display
   └─ debug-safe export

結論：

  • import/export 不應解析 secret-derived hyperlink text。
  • tooltip spellID/itemID 顯示可以保留，但必須和 runtime facts 分開。
  • debug export 要 escape pipe / hyperlink 類文字。

API Changes

API change summaries 是版本追蹤入口。

重要三層：

API changes
├─ Retail
│  ├─ 12.0.0
│  ├─ 12.0.1
│  ├─ 12.0.5
│  └─ 12.0.7
├─ change categories
│  ├─ Global API
│  ├─ ScriptObjects
│  ├─ Widgets
│  ├─ Events
│  └─ Structures
└─ EAM use
   ├─ removed API cleanup
   ├─ C_* migration
   └─ live validation targets

結論：

  • 每次 Retail patch 都要重新檢查 12.x API changes。
  • 12.0.7 API summary 已發布；仍需以實機驗證確認 EAM 行為。
  • 12.0.5 的 formatter / duration / table.freeze 已寫入 Docs/10_WARCRAFT_WIKI_12X_API_NOTES.md。

HOWTOs

HOWTOs 是教學頁索引，適合作為 UI 操作參考，但不是 Retail 12.x API contract。

重要三層：

HOWTOs
├─ Getting Started
│  ├─ Introduction to Lua
│  └─ Create a WoW AddOn
├─ UI examples
│  ├─ Making draggable frames
│  └─ frame examples
└─ EAM use
   ├─ options UI reference
   ├─ anchor movement
   └─ avoid example-grade hot path code

結論：

  • 可參考拖曳 frame 實作。
  • 不要直接複製 closure-heavy example 到 runtime hot path。
  • Options UI 可以保留簡單，避免 WeakAura-like system。

EAM Implementation Guide Summary

WoW API

WoW API is the main runtime API entry. EAM rewrite should give priority to finding the corresponding namespace from the API system instead of retaining the old global wrapper.

Three important layers:

WoW API
├─ API systems
│  ├─ C_UnitAuras
│  │  └─ AuraService
│  ├─ C_Spell
│  │  └─ CooldownService / SpellInfoService
│  ├─ C_Item
│  │  └─ ItemCooldownService
│  ├─ C_TooltipInfo
│  │  └─ low-frequency fallback only
│  └─ C_Secrets
│     └─ boundary checks

Conclusion:

  • 12.x only uses the Retail C_* namespace.
  • C_Secrets should be encapsulated centrally and not scattered in each UI function.
  • C_TooltipInfo is a fallback source, not a hot path fact source.

Lua API

Lua API Description WoW uses a Lua 5.1 subset and lacks the OS/file I/O library.

Three important layers:

Lua API
├─ basic functions
│  ├─ pcall / xpcall
│  └─ type / select
├─ string
│  ├─ string.match / gsub / format
│  └─ WoW custom string helpers
├─ table
│  ├─ table.create
│  ├─ table.wipe
│  ├─ table.freeze
│  └─ table.isfrozen

Conclusion:

  • string.match cannot be used with unsecured secret-derived tooltip text.
  • hot path avoids transient tables, table.insert, and large numbers of pairs.
  • table.freeze only works with immutable constants.

FrameXML API

FrameXML functions include UI helper, pool, EventUtil, TooltipUtil, LinkUtil, etc.

Three important layers:

FrameXML API
├─ Mixins / pools
│  ├─ CreateFramePool
│  ├─ CreateTexturePool
│  └─ CreateFontStringPool
├─ SharedXML
│  ├─ EventUtil
│  ├─ FrameUtil
│  └─ TooltipUtil
└─ UIParent / panels
   ├─ ShowUIPanel
   └─ Toggle*Frame

Conclusion:

  • EAM's UI/IconPool.lua can refer to the Blizzard pool pattern.
  • Do not call functions that open or close the protected panel in combat.
  • TooltipUtil can be used as a reference, but tooltip parsing must be low-frequency and secret-safe.

Widget API / Widget Scripts

Widget API and Widget script handlers are the basis of EAM Renderer, EventRouter, and Scheduler.

Three important layers:

Widget API
├─ Frame
│  ├─ RegisterEvent
│  ├─ RegisterAllEvents
│  └─ SetScript
├─ Cooldown
│  ├─ SetCooldownFromDurationObject
│  ├─ SetCountdownFormatter
│  └─ SetCountdownMillisecondsThreshold
└─ FontString
   ├─ SetText
   └─ ClearText

Widget scripts
├─ Frame
│  ├─ OnEvent
│  └─ OnUpdate
├─ Button
│  ├─ OnClick
│  └─ PreClick / PostClick
└─ EditBox
   ├─ OnTextChanged
   └─ OnEnterPressed / OnEscapePressed

Conclusion:

  • RegisterAllEvents is only suitable for debug/datamining, not EAM runtime.
  • EAM should have only one EventRouter OnEvent.
  • EAM should only have one Scheduler OnUpdate.
  • Cooldown should inherit DurationObject, do not let Lua count seconds every frame.

XML schema

XML schema Description XML can be used to build UIs, but the Lua Widget API can also do most things.

Three important layers:

XML schema
├─ Ui
│  ├─ Include
│  └─ Script
├─ LayoutFrame
│  ├─ Size
│  ├─ Anchors
│  └─ Animations
└─ Frame
   ├─ Layers
   ├─ Frames
   └─ Scripts

Conclusion:

  • The new architecture can keep the simple XML loader, but runtime icons should be given to IconPool.
  • Avoid creating a large number of global frame names in XML.
  • Avoid XML OnUpdate.

Events

Events / Category:API events is the event source for EventRouter.

Three important layers:

Events
├─ UNIT_AURA
│  ├─ unitTarget
│  ├─ updateInfo.addedAuras
│  ├─ updateInfo.updatedAuraInstanceIDs
│  └─ updateInfo.removedAuraInstanceIDs
├─ cooldown events
│  ├─ SPELL_UPDATE_COOLDOWN
│  ├─ SPELL_UPDATE_CHARGES
│  └─ BAG_UPDATE_COOLDOWN
└─ combat boundary
   ├─ PLAYER_REGEN_DISABLED
   └─ PLAYER_REGEN_ENABLED

Conclusion:

  • UNIT_AURA incremental data is the core entry of AuraService.
  • Removed aura instance ID can no longer check the complete aura, the service must first cache the security summary.
  • Out-of-combat refresh is the calibration point for boundary-limited state.

CVars

Console variables Description CVar can be managed using GetCVar / SetCVar / /console. Some secure CVar cannot be changed in combat.

Three important layers:

CVars
├─ query
│  ├─ GetCVar
│  ├─ GetCVarDefault
│  └─ C_CVar.GetCVarInfo
├─ mutation
│  ├─ SetCVar
│  └─ /console
└─ reset
   ├─ cvar_default
   └─ cvar_reset

Conclusion:

  • EAM should not rely on changing CVar to achieve functionality.
  • secret forced CVars should only be placed in live validation checklist.
  • Do not modify secure CVar in combat.

Macro commands are divided into character, combat and other commands.

Three important layers:

Macro commands
├─ character commands
├─ combat commands
│  ├─ cast / use
│  ├─ cancelaura
│  └─ startattack / stopattack
└─ addon slash commands
   ├─ /eam opt
   ├─ /eam debug
   └─ /eam export

Conclusion:

  • EAM slash commands can only be used for setting, detecting, debugging, and exporting.
  • No new combat automation is added.
  • Does not produce cast/use/cancelaura-like behavior.

Combat Log

Combat Log / COMBAT_LOG_EVENT_UNFILTERED are historically common AddOn sources, but 12.x targets with EAM rewrite should not rely on blind scanning.

Three important layers:

Combat Log
├─ Combat Log window / file
├─ COMBAT_LOG_EVENT_UNFILTERED
│  ├─ source
│  ├─ destination
│  ├─ prefix
│  └─ suffix
└─ EAM boundary
   ├─ not primary source
   ├─ no heavy parsing
   └─ use event services instead

Conclusion:

  • EAM for 12.x does not use CLEU as aura/cooldown primary source.
  • Only refer to it when necessary for debugging or non-hot path inspection.
  • Does not build WeakAura-like combat parser.

Conclusion:

  • import/export should not parse secret-derived hyperlink text.
  • tooltip spellID/itemID display can be retained, but must be separated from runtime facts.
  • debug export to escape pipe / hyperlink class literals.
