EventAlertMod 專家角色 RACI 定位職責矩陣

為了讓EAM重構專案中的AI專家智庫在協作開發時有章可循，並確保PR審查、履行優化與API安全防禦的責任邊界劃分，本文件正式專案的RACI矩陣。

後續所有的子代理派工、主代理整合、PR審查與代碼變更，都必須嚴格遵守這個RACI原則。

1. RACI 定義說明

  • R (Responsible - 負責執行者)：負責編寫程式碼、維護資料庫或編寫檔案的主要執行角色。
  • A（Accountable - 最終負責人）：此任務領域的設計決策、架構搭建性與計畫碼安全負有最終責任的角色。 每個任務領域只能有且有一個A，其可以對PR進行批准或否決。
  • C（諮詢 - 諮詢對象）：在設計、可行性評估、保密安全防禦或玩家實戰體驗上提供專業建議諮詢的角色。在執行任務時，必須主動向其獲取意見。
  • I (Informed - 通知對象)：任務或設定變更時的通知對象，感知細緻結果以保持各資訊模組同步。

EventAlertMod Expert role RACI Positioning responsibility matrix

In order to allow the AI expert think tank in the EAM reconstruction project to follow rules when collaborating on development, and to ensure the division of responsibilities between PR review, performance optimization, and API security defense, this document formalizes the RACI matrix of the project.

All subsequent sub-agent assignments, main agent integration, PR review and code changes must strictly abide by this RACI principle.

1. RACI Definition

  • R (Responsible - Responsible Executor): The main execution role responsible for writing code, maintaining databases, or writing files.
  • A (Accountable - Ultimate Responsible Person): The role that has the final responsibility for the design decisions, architectural construction, and project code security in this task area. Each task area can only have one A, which can approve or reject the PR.
  • C (Consulting - Consulting Object): A role that provides professional advice and consultation on design, feasibility assessment, confidentiality and security defense, or actual player experience. You must proactively seek their opinions when performing tasks.
  • I (Informed - Notification Object): Notification object when tasks or settings are changed, sensing detailed results to keep each information module synchronized.
